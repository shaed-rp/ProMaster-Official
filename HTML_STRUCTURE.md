# Full HTML Structure Documentation

**Project:** RAM ProMaster EV Landing Page  
**Framework:** Next.js 16.0.3 with App Router  
**Last Updated:** January 2025

---

## üìã Overview

This document describes the complete HTML structure generated by the Next.js application. The HTML is dynamically rendered server-side, with client-side hydration for interactive components.

---

## üåê How to View the Full Rendered HTML

### Option 1: View Source in Browser
1. Start the development server: `npm run dev`
2. Navigate to `http://localhost:3000` or `http://localhost:3000/promaster`
3. Right-click ‚Üí "View Page Source" (or press `Ctrl+U` / `Cmd+Option+U`)

### Option 2: Build and Inspect
1. Build the application: `npm run build`
2. Start production server: `npm start`
3. View source in browser or use curl: `curl http://localhost:3000`

### Option 3: Use Browser DevTools
1. Open browser DevTools (F12)
2. Go to Elements/Inspector tab
3. Right-click on `<html>` ‚Üí "Copy" ‚Üí "Copy outerHTML"

## üìù Notes

1. **Dynamic Content**: Most content is loaded from `data/promasterData.json`
2. **Server-Side Rendering**: Static content is rendered on the server
3. **Client-Side Hydration**: Interactive components are hydrated client-side
4. **Responsive**: HTML structure adapts to different screen sizes
5. **SEO Optimized**: Comprehensive meta tags and structured data
6. **Accessible**: WCAG 2.1 AA compliant with proper ARIA attributes
